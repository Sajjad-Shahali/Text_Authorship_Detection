# MALTO Recruitment Hackathon — Text Authorship Detection

> **Author:** [Sajjad Shahali](https://github.com/sajjad-shahali)
> **Competition:** Kaggle MALTO Recruitment Hackathon
> **Task:** 6-class multiclass text classification
> **Metric:** Macro F1 Score
> **Constraint:** Only the provided dataset — no external data, no pre-trained AI detectors

---

## What This Project Does

Given a text sample, the model decides:

1. Was this written by a **human** or generated by an **AI model**?
2. If AI-generated, **which specific model** produced it?

| Label | Source        |
|-------|---------------|
| 0     | Human-written |
| 1     | DeepSeek      |
| 2     | Grok          |
| 3     | Claude        |
| 4     | Gemini        |
| 5     | ChatGPT       |

Evaluation uses **Macro F1 Score** — all 6 classes are weighted equally.

## Running the Pipeline

### Train

```bash
python main_train.py --config configs/config.yaml
```

```bash
make train
```

What happens:
1. Loads config and creates a timestamped experiment folder under `artifacts/experiments/`
2. Loads and validates `data/train.csv`
3. Runs 5-fold stratified cross-validation for all configured models
4. Saves model comparison to `artifacts/metrics/model_comparison.csv`
5. Trains final pipeline on all data → `artifacts/models/best_model.joblib`

### Inference

```bash
python main_infer.py --config configs/config.yaml
```

```bash
make infer
```

Loads the trained pipeline, runs it on `data/test.csv`, saves predictions.

### Generate Submission

```bash
python main_submit.py --config configs/config.yaml
```

```bash
make submit
```

Produces and validates `artifacts/submissions/submission_latest.csv`.
Upload this file to the Kaggle competition page.

### Run Tests

```bash
make test
```

```bash
make test-cov     # with coverage report
```

All 33 tests use synthetic data — no competition files required.

## Approach

### Feature Engineering

Two TF-IDF representations are combined using `FeatureUnion`:

| Vectorizer    | Analyzer | n-gram range | Key signal                           |
|---------------|----------|-------------|--------------------------------------|
| Word TF-IDF   | word     | (1, 2)      | Vocabulary, phrase-level style       |
| Char TF-IDF   | char_wb  | (3, 5)      | Punctuation, spacing, morphology     |

Character n-grams are especially powerful for detecting LLM output — AI models have very distinct punctuation and spacing patterns.

### Models Compared

| Model              | Key config           | Notes                              |
|--------------------|----------------------|------------------------------------|
| Logistic Regression| C=1.0, solver=lbfgs  | Probabilistic, well-calibrated     |
| LinearSVC          | C=0.1                | Fast on high-dimensional sparse    |
| ComplementNB       | alpha=0.1            | Robust for imbalanced classes      |

The best-performing model (by CV Macro F1) is selected for the final training run and submission.

### Validation

- **Stratified 5-fold cross-validation** — class balance preserved in every fold
- **Primary metric:** Macro F1 (not accuracy)
- Per-fold train/val F1 logged — overfit gap flagged if > 0.10
- Out-of-fold predictions used for full error analysis

### Anti-Leakage Guarantee

The complete pipeline (`Preprocessor → FeatureUnion → Classifier`) is **rebuilt and fitted inside each CV fold**. No vectorizer state can bleed from validation data into training features.

---

## Overfitting Prevention

| Safeguard                   | How                                   |
|-----------------------------|---------------------------------------|
| Vectorizer inside CV folds  | Full pipeline rebuilt per fold        |
| Regularization              | `C` parameter in LR and LinearSVC     |
| Vocabulary size cap         | `max_features=100_000`                |
| Term frequency thresholds   | `min_df=2`, `max_df=0.95`             |
| Stratified splits           | Class balance maintained              |
| Train vs val gap monitoring | Logged per fold, warning if gap > 10% |

## Limitations

- TF-IDF cannot capture semantic meaning or long-range coherence
- Linear classifiers may miss subtle interaction patterns between features
- ComplementNB assumes feature independence (violated in practice)
- No held-out test set used during model selection — risk of CV optimism
