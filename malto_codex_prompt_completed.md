
# MALTO Recruitment Hackathon – Engineering Brief and Codex Prompt (Enhanced)

## Project Overview

This project targets the **Kaggle MALTO Recruitment Hackathon** competition.

The task is a **6‑class text classification problem** where the model must determine:

1. Whether a text was **written by a human** or **generated by an AI model**
2. If AI‑generated, **which specific model produced it**

### Labels

| Label | Source |
|-----|-----|
|0|Human-written text|
|1|DeepSeek-generated|
|2|Grok-generated|
|3|Claude-generated|
|4|Gemini-generated|
|5|ChatGPT-generated|

### Key technical challenges

- Distinguishing **human vs AI** text when LLMs generate very fluent content
- Differentiating **between different LLM families**
- Avoiding **overfitting on stylistic artifacts**
- Optimizing **Macro F1**, not just accuracy

Macro F1 means every class must perform well.

# Data Preprocessing

Preprocessing must be **minimal**.

Over-cleaning removes stylistic signals useful for classification.

Configurable options:

- unicode normalization
- whitespace normalization
- optional lowercasing
- optional punctuation preservation
- optional number preservation
- removal of repeated spaces

Default behavior: **minimal cleaning**.

---

# Feature Engineering

Implement TF‑IDF features.

Support:

- word n‑grams
- character n‑grams

Recommended baseline:

```
Word TFIDF: ngram_range=(1,2)
Char TFIDF: ngram_range=(3,5)
```

Combine using FeatureUnion or equivalent.

# Validation Strategy

Use **Stratified K-Fold cross-validation**.

Required outputs:

- Macro F1
- Accuracy
- Per-class precision/recall/F1
- Confusion matrix

Artifacts saved:

```
artifacts/metrics/cv_results.json
artifacts/metrics/classification_report.txt
artifacts/metrics/confusion_matrix.csv
```

---

# Overfitting Prevention

Required safeguards:

- Stratified cross validation
- Vectorizers fitted **inside CV folds**
- Regularized models
- Hyperparameter tuning
- Monitoring train vs validation metrics
- Detecting high fold variance

Hyperparameters to tune:

- C
- min_df
- max_df
- ngram ranges
- max_features
